### 需求文档：





3D场景语义平面提取与轻量化重建系统 - 项目需求文档

1\. 项目概述

本项目旨在开发一个基于 Web 的 3D 几何处理工具。系统需要读取由 MapAnything 等扫描算法生成的超高面数 3D 室内场景模型（如 test\_scene.glb），通过底层算法自动剥离并提取出场景中的语义平面（墙体、地面、天花板），最终将其转化为极简的低模（Low-poly Mesh）。

这些低模将用于后续的 Web 3D 交互（如同特效合成般在原场景上拖拽摆放虚拟家具、替换材质等）。



2\. 系统架构选型

系统采用前后端分离架构，以保证前端交互的流畅性和后端计算的性能：



前端 (Frontend): HTML5 + CSS3 + JavaScript + Three.js（负责 3D 场景渲染、UI 控件交互、低模叠加预览）。



后端 (Backend): Python + FastAPI (或 Flask) + Open3D + NumPy（负责接收前端参数，执行繁重的点云降采样、RANSAC 拟合及几何重拓扑）。



数据交换: JSON (传输参数) + Base64 编码的 GLB/OBJ 字符串 (传输生成的低模数据)。



3\. 前端交互与 UI 需求

界面需分为左右两部分：左侧为 3D 视口，右侧为控制面板。



3D 视口 (Viewport):



初始化时加载并显示根目录下的 test\_scene.glb（建议使用半透明或线框材质展示原高模，以便透视内部）。



接收到后端返回的低模后，将其精确叠加在原场景上，并按语义进行颜色区分：地面=绿色，天花板=红色，墙面=蓝色。



控制面板 (Control Panel):



滑块 1：体素大小 (Voxel Size) - 范围 \[0.01, 0.2]，默认 0.05。控制点云降采样的粗细。



滑块 2：距离阈值 (Distance Threshold) - 范围 \[0.01, 0.2]，默认 0.05。控制 RANSAC 算法识别平面的厚度包容度。



滑块 3：角度容差 (Angle Tolerance) - 范围 \[1.0, 15.0]，默认 5.0。单位为度，用于判断平面是否严格垂直或水平。



按钮 1：运行提取 (Run Extraction) - 点击后锁定 UI，展示 Loading 状态，并将滑块参数发送至后端。



按钮 2：导出低模 (Download Low-Poly) - 将提取出的干净墙体/地面数据作为 .glb 或 .obj 文件下载到本地。



4\. 后端核心算法管线 (Algorithm Pipeline)

后端需提供一个 API 接口（如 /api/extract），按以下顺序执行几何处理：



数据加载与轻量化 (Preprocessing):



利用 Open3D 加载 test\_scene.glb，将其转换为点云格式。



根据前端传入的 Voxel Size 进行体素降采样，以大幅降低计算量。



计算点云法线，并统一法线朝向。



序列化平面提取 (Sequential RANSAC):

\*



使用 while 循环不断执行 RANSAC 算法提取最大平面。



每次提取后，将属于该平面的点从主点云中剔除，再进行下一次提取。



停止条件：提取满 10-15 个平面，或剩余点云数量低于总量的 5%。



语义分类与打标 (Semantic Classification):



地面 (Floor): 法线方向与 Y 轴平行（误差在 Angle Tolerance 内），且空间位置（质心 Y 坐标）最低的水平面。



天花板 (Ceiling): 法线方向与 Y 轴平行，且空间位置最高的水平面。



墙面 (Walls): 法线方向与 Y 轴垂直（点乘接近 0，误差在 Angle Tolerance 内）的非水平面。



二维投影与低模生成 (Retopology):



将每个语义平面的三维散点投影到其自身的局部二维平面上。



计算二维边缘轮廓，并反向拉伸/生成极简的 3D 多边形网格（Low-poly Mesh）。



5\. 工程鲁棒性与异常处理 (Edge Cases \& Fallbacks)

为防止真实扫描数据导致算法崩溃或提取错误，后端代码必须包含以下防御性逻辑：



物理常识过滤 (防误判): \* 需剔除面积过小（阈值可设为 <1m 

2

&nbsp;）的无效平面碎片。



若水平平面的高度处于 0.5m∼1.2m 之间且法线朝上，大概率为桌面或床铺，需将其从“地面”或“墙面”的分类中剔除。



重力方向自校准 (防倾斜):



切勿绝对信任扫描模型的原始 Y 轴。在语义分类前，应先找到面积最大的正交平面组，计算出真实的物理“上”方向（Up-vector），并据此校准后续的角度计算。



复杂墙体适配 (防穿模):

\*



在步骤 4 生成低模时，避免使用简单的 2D OBB（包围盒），必须采用 Alpha Shape (凹包) 算法计算边界。确保生成的低模能完美避开 L 型墙角或非规则顶部的镂空区域。



通信防阻塞:



后端接口返回给前端的数据，必须是经过步骤 4 生成的低模顶点/索引，严禁直接返回几十万个点的原始点云数据，防止浏览器内存溢出（OOM）。
